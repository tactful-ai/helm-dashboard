Data Flow

1. Web app

graph TD; Helm-CLI-->Express-API-->API-Connector-->Vue-App;

2. Desktop app

graph TD; Helm-CLI-->Electron-server-->Electron-client-->background.js-->Electron-API-->Vue-App;

Important Classes

1. helm (backend/cmd/helm)

Communicates with helm-cli

2. kubectl (backend/cmd/kubectl)

Communicates with keubectl-cli

3. API Connector (frontend/utils/API-Connector)

Communicates with our express API by fetching data from it

4. HelmMainController (desktop/src/electron-client)

Returning data comming from the helm-cli from the render process of electron, to the main process

5. ElectronAPI (desktop/src/ElectronAPI)

1. background.js communicates with electron-server (helm class) to retreive data from it, and returns it to electron-client (HelmMainController class)

2. ElectronAPI communicates with electron-client (HelmMainController class) to get the data from it, and parse it.
